drt.remote

      Run WLW, Lithium, P13N, WLI and checkintest.

remote.all:

      runs all tests that make up last_clean_test_platform_ except for remoteDRT_buildkit which takes too long.

remote.everything:

      Run every possible remote tests. Note that some of these tests are not known to be clean at last_clean_test_platform_<branch>. In fact, since the CruiseControl instances for all these tests are not synchronized there is no single change number that guarantees that all the tests will be clean. The tests that are known clean are run first and if the later on, unknown tests fail the remote test will note that the test is not known to be clean.

remote.multiple:

      Run multiple remote tests. This is probably most useful to make a list of tests for auto-submit or so that the tests will only use one machine serially rather than running each remote test in parallel on a group of machines.

remote.status

      Show the status of remote tests.

remoteDRT_all

      Run deploy.checkintest, core.checkintest and wsee.checkintest.
